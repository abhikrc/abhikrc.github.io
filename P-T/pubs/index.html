A. Five Selected Publications

In the following, I present the summary of five selected publications of mine. The publications are in the domain of program analysis (ICSE 2004, Real-time Systems Journal 2006) or formal models and methods (ICSE 2006, FM 2006, CAV 2001). I have chosen the publications for the originality of the proof methods or analysis methods therein, as well as their applicability to real-world problems. Two of the five publications mentioned in the following have led to large-scale software analysis tools for standard programming languages. In particular, my publication in ICSE 2004 has led to the Jslice dynamic slicing tool for Java programs, and the publication in Real-time Systems Journal 2006 has led to the Chronos execution time analysis tool for C programs.

 

Interacting  Process Classes, International Conference on Software Engineering (ICSE)  2006.

This work develops a novel modeling and simulation framework for distributed embedded systems with many similar interacting processes. The main contribution is to support time and memory efficient simulation of designs while allowing combination of rich modeling constructs compatible with the Unified Modeling Language (UML). The applications of this work lie in the design of distributed embedded systems with behaviorally similar processes such as a traffic controller controlling many cars/trains/aircrafts

 

Using Compressed Bytecode Traces for Slicing Java programs, International Conference on Software Engineering (ICSE)  2004.

This work develops methods for debugging Java programs based on analysis of program execution runs. The research has led to the Jslice dynamic slicing tool for Java programs --- the first of its kind. This is useful, since dynamic slicing is a popular technique which forms the core of many program development/analysis methods. However, slicing tools for real-life programming languages (like Java) are relatively hard to come by. Our research in this paper aims to fill this gap.

 

Modeling Out-of-order processors for WCET Analysis, Real-Time Systems Journal, 2006

This work develops static analysis methods for tightly estimating the maximum execution time of a program when the program is executing on a processor with advanced micro-architectural features. The work led to the Chronos Worst-case Execution time estimation tool for C programs, which can be used for scheduling and performance estimation of real-time/embedded software. The typical applications are in safety-critical embedded software from application domains like automotive and avionics.

 

A Memory Model Sensitive Checker for C#, International Symposium on Formal Methods (FM) 2006.

Reasoning about concurrent program behaviors is hard. In this work, we study how to accurately capture all possible behaviors of a program written in a modern multi-threaded programming language like C#. We show that current verification techniques (mistakenly) assume a strong execution model called  "Sequential Consistency", which may result in a verified correct program exhibiting violations of the property verified! We then discuss how the semantics of multi-threading can be accurately described and incorporated into program verification. Even though the results are for C#, the ideas are generic and can be applied to other programming languages, such as Java.

 

Automated inductive verification of Parameterized Protocols, International Conference on Computer Aided Verification (CAV) 2001.

Many distributed systems can be described as unbounded collections of similar processes. In this work, we prove properties about such systems by employing a mix of automated methods (search) and mathematical induction. The main conceptual novelty comes from automating these induction proofs by gleaning the induction schema directly from the system description; logic program manipulation techniques are used for this purpose. A large part of this research was done during my Ph.D. work at SUNY Stony Brook.

 

 

B.  Usage Statistics for the JSlice tool developed as part of my research.

In my research on software validation, I have developed program analysis tools for widely used programming languages like Java. In particular, together with my PhD student Wang Tao, I have developed the first dynamic slicing tool for Java, JSlice. Dynamic slicing is a popular technique for program debugging and understanding. Prior to our work, there was no dynamic slicing tool for real-life programming languages like Java. The research work done in building JSlice has been reported in the following papers, and the tool is available from  http://jslice.sourceforge.net

Dynamic Slicing on Java Bytecode Traces
Tao Wang and Abhik Roychoudhury
ACM Transactions on Programming Languages and Systems (TOPLAS), To appear.

Using Compressed Bytecode Traces for Slicing Java Programs
Tao Wang and Abhik Roychoudhury
ACM/IEEE International Conference on Software Engineering (ICSE) 2004.
