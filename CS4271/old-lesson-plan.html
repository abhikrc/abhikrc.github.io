Week 4, 6 February Temporal Logics  We will continue with the material given for the previous week. The Computation Tree Logic (CTL) will be covered in this lecture. The textbook derives CTL as a sub-logic of CTL*. An alternative view is given in the book Logic in Computer Science (Huth and Ryan, RBR in Science Library);  in this book the syntax and semantics of CTL are developed directly.

Week 5, 13 February SMV Model Checker  In this lecture, we will familiarize ourselves with SMV's modeling language. We will look at toy examples at first (such as the Alternating Bit Protocol). Later on, in the revision hour I will share some past experiences in verifying real-life designs --- the AMBA bus protocol from ARM --- using SMV. At this point we have not learnt the internal algorithms (model checking) of SMV. However, this knowledge is not essential to get acquainted with using SMV as a verification tool.

Week 9, 20 March Binary Decision Diagrams - I   In this lecture, we will discuss Binary Decision Diagrams, a compact data structure for representing boolean functions. The connection of this data structure to our course comes from using BDDs to represent the state space being verified. In a subsequent lecture we will see how the sets of states and transitions of a Kripke Structure can be modeled as boolean functions as hence as BDDs.

A bit of History: The concept of BDD is a very old one, but the Reduced Ordered BDDs that we will discuss in this lecture was first proposed by Randy Bryant in 1985-86. His original paper is available from his webpage, see http://www-2.cs.cmu.edu/~bryant/pubdir/ieeetc86.pdf

Week 10, 27 March

Binary Decision Diagrams - II We will continue with our discussion on BDDs. In this lecture, we will set the stage for using BDDs in model checking. This will be done by discussing how BDDs can represent sets of states and transitions. We will also discuss how set operations can be preformed via boolean operations on sets of states represented as BDDs.

Week 11, 3 April Symbolic Model Checking We will discuss a symbolic model checking algorithm for the Computation Tree Logic (CTL). In this algorithm, sets of states and transitions will be represented as Binary Decision Diagrams, and the effect of graph traversal will be achieved implicitly via logical operations.
