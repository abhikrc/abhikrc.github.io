1.  Modeling and Validation of  System-on-Chip Bus Protocols.

A typical system-on-chip (SoC) embedded system consists of a number of  functional modules, such as one or more CPU/signal processors, high performance peripherals, DMA controllers, on-chip memory and interfaces implemented on a single chip. Therefore we need an on-chip bus protocol to interconnect SoC modules. Typically, the description of these protocols is given as an informal open specification to the the vendors who implement it.  However, precise modeling and analysis of these protocols is important for two reasons. First of all, an informal description is subject to different interpretation by different vendors. Secondly, since these protocols are implemented in a single chip, it is difficult to perform conventional system level testing (e.g. via a logic analyzer which snoops on the bus). Hence it is more important to verify crucial design invariants in the model of the protocol. The aim of these group of projects will be to study the various existing system-on-chip bus protocols and find out the similarities/differences between these protocols.  Existing protocols include :  AMBA, VIA, PI-Bus, CoreConnect .

Possible projects :
(a) Developing a common architecture for system-on-chip bus protocols.
(b) Combination of model checking and testing specifically for these bus protocols
(c) Modeling and verification of a abstract version of a specific bus protocol.

2. Formal Verification of Simple Embedded Processors

The  technique of model checking is now widely used in the processor
design industry. In particular, industry leaders such as Intel have research
and developemnt teams dedicated to verification of Intel processors.
Processor verification is a relatively mature technology with lot of resources
available on techniques for state-space reduction. In this project, you will
verify units of a real-life embedded processor LEON (specification is
available freely).

3. Formal Validation of (distributed) controllers

Currently, there is a focus on developing embedded systems starting from high-level requirements. In this context, UML diagrams form a suitable choice. Of specific interest are StateCharts and Message Sequence Charts. StateCharts are suitable for modeling the components of a system, whereas the interactions between the components is given by Message Sequence Charts. You  may choose to take  a specification of a concrete protocol/ hardware unit and try to verify them using a model checking tool
